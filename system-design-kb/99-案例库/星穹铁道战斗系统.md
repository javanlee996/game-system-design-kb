# 案例：崩坏：星穹铁道的战斗系统

> **涉及模块**：01-战斗系统、02-成长系统、06-商业化设计

---

## 1. 背景

- **游戏类型**：回合制RPG
- **开发商**：米哈游
- **上线时间**：2023年4月
- **商业表现**：首月流水预估超10亿人民币

**设计挑战**：
如何在2023年让"回合制"这个被认为"过时"的品类焕发新生？

---

## 2. 系统拆解

### 2.1 核心战斗循环

```
角色速度值 → 行动顺序条 → 玩家选择行动 → 技能释放 → 
伤害/治疗/增益 → 削韧计算 → 击破状态触发 → 回到行动条
```

### 2.2 关键机制详解

#### 机制A：速度条系统（ATB变体）

| 要素 | 设计细节 |
|------|----------|
| **可视性** | 屏幕左侧显示完整的行动顺序队列 |
| **可干预性** | 拉条（提前行动）、推条（延后行动）、冻结（跳过回合） |
| **策略深度** | 玩家可以计算："如果我此刻使用X技能，敌人将在Y回合后行动" |

**设计意图**：将传统回合制"你拍一我拍一"的静态节奏，改造为**可操控的动态节奏**。

#### 机制B：韧性-击破系统

```
敌人血条上方有韧性条 → 
使用对应属性攻击削减韧性 → 
韧性归零进入"弱点击破"状态 → 
击破者获得额外行动回合 + 敌人受到额外伤害 + 属性特效
```

**设计亮点**：
- 创造了"次要目标"——不只是打血，还要破盾
- 属性克制从"伤害加成"变为"机制触发"，策略意义更强
- 击破的延迟收益（击破后敌人受到更多伤害）创造了决策张力

#### 机制C：战技点（Skill Point）

- 全队共享的资源
- 普攻回复1点，战技消耗1点，大招独立于战技点

**设计意图**：
- 防止玩家无脑按最强技能（资源约束）
- 创造"普攻-战技"的节奏交替
- 角色分工：有的角色负责产点，有的角色负责消耗

## 3. 与其他系统的耦合

### 3.1 与成长系统的耦合

| 成长维度 | 对战斗的影响 |
|----------|--------------|
| **角色等级** | 基础属性提升 |
| **光锥（武器）** | 改变角色定位（如把输出手变成辅助） |
| **遗器（圣遗物）** | 速度属性直接影响行动顺序，是策略核心 |
| **星魂（命座）** | 部分星魂改变机制（如额外战技点回复） |
| **行迹（技能树）** | 解锁新的能力分支 |

**关键洞察**：速度属性在星铁中比在原神中重要得多，因为**速度直接=行动次数=输出机会**。

### 3.2 与商业化系统的耦合

- 新角色往往带来**新机制**（如新属性、新击破效果）
- 深渊（忘却之庭）的环境BUFF轮换，鼓励玩家抽取当期UP角色
- 但0命角色机制完整，高命主要是数值提升（相对克制的商业化）

## 4. 优劣评估

### ✅ 做得好的地方

1. **回合制的现代化**
   - 保留了回合制的策略深度
   - 加入了即时反馈（击破特效、连携攻击）
   - 节奏比传统回合制快得多

2. **策略分层**
   - 表层：哪个亮了点哪个（新手友好）
   - 中层：属性克制、战技点管理（普通玩家）
   - 深层：速度阈值计算、行动条操控（硬核玩家）

3. **视觉与机制的融合**
   - 大招动画不能跳过——强化角色价值感知
   - 击破特效、弱点爆炸提供强烈正向反馈

### ❌ 存在的问题

1. **自动战斗的悖论**
   - 低难度内容太简单，自动战斗即可
   - 高难度内容太复杂，AI操作智障
   - 导致玩家"手操嫌麻烦，自动打不过"的挫败感

2. **属性膨胀的隐忧**
   - 新角色机制越来越复杂，老角色容易被淘汰（数值和机制双重膨胀）
   - 长线运营的压力

3. **PVE内容枯竭**
   - 战斗系统深度依赖PVE
   - 内容消耗速度远超生产速度

## 5. 可迁移经验

### 可以搬到其他游戏的设计

| 经验 | 适用场景 |
|------|----------|
| **可视化行动顺序** | 任何回合制或半即时制游戏 |
| **次要目标设计**（韧性条） | 需要增加战斗维度的ARPG |
| **全队共享资源**（战技点） | 多角色队伍构建的游戏 |
| **机制分层的策略深度** | 所有需要兼顾新手和硬核玩家的游戏 |

### 有特定语境限制的设计

- **无法跳过大招动画**：星铁作为内容型游戏可行，竞技游戏绝对不行
- **属性击破机制**：需要固定队伍搭配的游戏才能支撑，单角色游戏不适用
- **命座系统**：F2P手游商业模式，买断制游戏需要改造

## 6. 设计模板提取

基于星铁战斗系统，可以提取以下设计模板：

### 模板：回合制现代化改造

```
1. 识别传统玩法的"慢点"（回合制=等待时间长）
2. 增加可干预性（让玩家能操控节奏）
3. 增加次要目标（不只是打血）
4. 策略分层（新手友好，老手有深度）
5. 强视觉反馈（让等待也有爽感）
```

### 模板：资源约束设计

```
1. 识别要约束的行为（无脑放技能）
2. 设计共享/个人资源系统
3. 创造"产出者"和"消费者"的角色分工
4. 测试：资源紧张时玩家是否还有选择空间？
```
