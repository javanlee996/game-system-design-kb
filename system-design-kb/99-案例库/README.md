## 案例分析格式说明

每个案例按以下结构展开：

```markdown
### 1. 背景
- 游戏类型、目标用户、商业表现

### 2. 系统拆解
- 核心循环图
- 关键机制详解

### 3. 设计意图分析
- 设计者要解决什么问题？
- 为什么选择这种方案？

### 4. 优劣评估
- 做得好的地方
- 存在的问题

### 5. 可迁移经验
- 什么可以搬到其他游戏？
- 什么有特定语境限制？
```
