# 06-商业化设计

> 商业化设计不是"怎么让玩家花钱"，而是"如何让玩家愿意长期投入"——无论是时间还是金钱。

## 一、F2P商业模式基础

### 1.1 核心公式

```
收入 = 用户量 × 付费率 × ARPU（每用户平均收入）
```

F2P游戏的设计目标是：
- 最大化用户量（低门槛）
- 维持合理的付费率（通常2-5%）
- 最大化ARPU（ whale 玩家贡献大部分收入）

### 1.2 鲸鱼理论（Whale Theory）

| 玩家类型 | 占比 | 贡献收入 | 设计策略 |
|----------|------|----------|----------|
| **免费玩家** | 90-95% | 0% | 提供完整体验，维持社群热度 |
| **小额付费** | 3-7% | 10-20% | 月卡、战令等固定投入 |
| **中额付费** | 1-2% | 20-30% | 角色/装备收集 |
| **鲸鱼** | 0.1-0.5% | 50-70% | 极限养成、排行榜、社交地位 |

**争议性**：鲸鱼模式被批评为"剥削成瘾者"，但在F2P行业普遍存在。

### 1.3 免费玩家的价值

免费玩家虽然不付费，但提供了：
- 社群活跃度
- PVP中的"内容"（给付费玩家击败）
- 口碑传播
- 未来转化潜力

**设计原则**：免费玩家应该能体验大部分内容，只是进度慢一些。

## 二、付费点设计

### 2.1 付费点类型

| 类型 | 说明 | 案例 |
|------|------|------|
| **便利性** | 省时间 | 体力购买、加速券 |
| **外观** | 不影响数值 | 皮肤、坐骑、装饰 |
| **内容** | 解锁新玩法 | DLC角色、新剧情 |
| **数值** | 直接变强 | 强化材料、高级装备 |
| **概率** | 赌运气 | 抽卡、开箱子 |
| **特权** | 身份象征 | VIP等级、称号 |

### 2.2 付费深度设计

**付费深度** = 一个玩家理论上能花多少钱

| 游戏类型 | 付费深度 | 案例 |
|----------|----------|------|
| 休闲游戏 | ¥0-100 | 三消、放置 |
| 中核游戏 | ¥100-5000 | 原神、星铁 |
| 重核游戏 | ¥5000+ | 梦幻西游、传奇类 |

**设计考量**：
- 深度太浅：鲸鱼没有花钱空间
- 深度太深：免费玩家感到绝望

### 2.3 付费节奏

**首次付费（First Purchase）**：
- 通常设计极低门槛（¥6、¥1）
- 目的：打破心理防线，让玩家习惯付费

**持续付费**：
- 月卡：固定收入，绑定日活
- 战令：阶段性目标，周期性付费
- 活动：限时刺激，FOMO（错失恐惧）

## 三、抽卡/Gacha设计

### 3.1 抽卡的心理机制

抽卡利用了多重心理效应：

| 效应 | 说明 |
|------|------|
| **可变比率强化** | 不确定的奖励比确定的更上瘾 |
| **近失效应** | "差一点就中了"比"完全没中"更诱人 |
| **沉没成本** | "已经抽了50发了，再抽10发就保底" |
| **从众效应** | 看到别人出货，自己也想试试 |
| **初始效应** | 新手往往有更高的"运气"（实际可能是保底） |

### 3.2 保底机制

现代抽卡游戏必须有保底，否则玩家会在极端非酋情况下流失。

| 保底类型 | 说明 | 案例 |
|----------|------|------|
| **软保底** | 概率逐渐提升 | 原神73发后概率提升 |
| **硬保底** | 固定次数必出 | 原神90发必出5星 |
| **大小保底** | 第一次保底50%UP，第二次100%UP | 原神、星铁 |
| **保底继承** | 池子结束保底次数保留 | 原神角色池 |
| **保底兑换** | 抽卡副产物可兑换目标 | 星铁星芒商店 |

### 3.3 抽卡定价的心理锚定

| 定价策略 | 说明 |
|----------|------|
| **首充双倍** | 第一次充值性价比最高，打破心理防线 |
| **礼包比直充划算** | 让玩家觉得"赚到了" |
| **限时折扣** | 创造紧迫感 |
| **大保底预期** | 让玩家提前准备好180抽的钱 |

## 四、Battle Pass设计

Battle Pass（战斗通行证/战令）是近年来最成功的商业化创新之一。

### 4.1 核心机制

```
购买通行证（通常¥68）→ 
通过完成任务获得经验 → 
升级解锁奖励 → 
通常包含价值远超售价的资源
```

### 4.2 为什么有效？

| 因素 | 说明 |
|------|------|
| **沉没成本** | 买了之后不玩就亏了，绑定日活 |
| **超值感** | 展示总价值远超售价 |
| **进度可视化** | 清晰的等级和奖励预览 |
| **FOMO** | 赛季结束就绝版 |

### 4.3 设计要点

- **免费线**：让免费玩家也能获得一些东西
- **付费线**：让付费玩家感到"赚到了"
- **终极奖励**：赛季末的高价值奖励，激励完成
- **任务设计**：不能太简单（太快完成），也不能太难（挫败）

## 五、伦理与长期主义

### 5.1 争议性设计

| 设计 | 争议 |
|------|------|
| **无限抽卡** | 是否等同于赌博？ |
| **鲸鱼模式** | 是否剥削成瘾者？ |
| **pay to win** | 付费玩家碾压免费玩家，是否公平？ |
| **黑暗模式** | 故意 confusing 的UI让玩家误操作消费 |
| **儿童诱导** | 明亮色彩、卡通角色吸引儿童消费 |

### 5.2 各国监管

- **中国**：抽卡必须公示概率，必须有保底，未成年人限时
- **日本**：完全扭蛋禁止（必须有保底）
- **比利时/荷兰**：认为抽卡是赌博，部分游戏下架
- **欧盟**：正在讨论统一监管

### 5.3 长期主义视角

**短视做法**：
- 透支玩家信任，杀鸡取卵
- 数值膨胀逼迫付费
- 新玩家无法追赶

**长期做法**：
- 免费玩家也能有完整体验
- 付费主要是"省时间"和"收藏"
- 持续产出优质内容

**案例对比**：
- **正面**：原神（0命完整，付费主要是收集和省事）
- **负面**：某些传奇类游戏（不付费就是沙包）

## 六、案例分析

### 6.1 原神：克制的商业化

**设计特点**：
- 0命角色机制完整，高命主要是数值
- 武器池毒但可选（4星武器也能通关）
- 深渊（最高难度内容）难度适中，0氪可满星
- 付费点主要是"收集"和"省事"

**结果**：
- 极高的玩家口碑
- 极高的付费转化（因为玩家愿意支持）
- 证明"良心"也能赚钱

### 6.2 星穹铁道：继承与优化

**相比原神的优化**：
- 光锥（武器）池单UP，更容易抽到
- 模拟宇宙提供免费的5星光锥
- 遗器（圣遗物）虽然随机，但定向道具降低随机性

**核心逻辑一致**：
- 0命完整体验
- 付费=省时间+收藏

## 七、设计模板

### 模板1：付费点检查

```
1. 列出所有付费点
2. 对每个付费点问自己：
   □ 玩家为什么愿意为此付费？（解决了什么痛点）
   □ 免费玩家是否有替代方案？
   □ 这个付费点是否会被骂"逼氪"？
3. 检查付费深度：
   □ 一个小额付费玩家能花多少钱？
   □ 一个鲸鱼能花多少钱？
   □ 是否存在"不付费没法玩"的内容？
```

### 模板2：抽卡系统检查

```
1. 概率公示：
   □ 各星级概率是否清晰展示？
   □ UP角色概率是否明确？
2. 保底机制：
   □ 软保底从第几发开始？
   □ 硬保底是多少发？
   □ 保底是否继承？
   □ 是否有兑换机制？
3. 心理预期管理：
   □ 玩家是否能计算"我要准备多少钱"？
   □ 非酋体验是否不会太差？
```

### 模板3：Battle Pass设计检查

```
1. 价格是否合理？（通常¥30-128）
2. 免费线是否有价值？
3. 付费线总价值是否远超售价？（通常5-10倍）
4. 任务是否多样化？（不要全是"打X场"）
5. 进度节奏是否合理？（不要太快满级，也不要满不了）
6. 终极奖励是否有吸引力？
```

---

*> 商业化设计是一门平衡的艺术：让玩家感到"值得"，而不是"被迫"。*
