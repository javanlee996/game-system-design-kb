## 一、成长系统的分类

### 1.1 按成长维度

| 类型 | 说明 | 代表游戏 |
|------|------|----------|
| **数值成长** | 攻击力、生命值等硬属性 | 几乎所有RPG |
| **机制成长** | 解锁新技能、新机制 | 艾尔登法环、哈迪斯 |
| **技巧成长** | 玩家自己变强（非角色） | 只狼、鬼泣 |
| **社交成长** | 排名、称号、外观 | 魔兽世界、FF14 |

### 1.2 按成长节奏

| 类型 | 周期 | 特征 |
|------|------|------|
| **即时成长** | 秒级 | 打怪立刻获得经验 |
| **短期成长** | 分钟-小时级 | 完成副本获得装备 |
| **中期成长** | 天-周级 | 角色升级、装备强化 |
| **长期成长** | 月-年级 | 毕业装备、全收集 |

### 1.3 按确定性

| 类型 | 说明 | 案例 |
|------|------|------|
| **确定性成长** | 投入必有回报 | 经验值升级、材料合成 |
| **随机性成长** | 投入结果不确定 | 原神圣遗物、抽卡 |
| **混合型** | 保底+随机 | 星铁行迹、命座 |

## 二、核心设计原则

### 2.1 可见的进度

**问题**：玩家需要感知到"我在变强"。

**设计方法**：
- **数值可视化**：伤害数字、血量条、等级数字
- **里程碑设计**：每10级一个大提升，每5级一个新技能
- **对比参照**：打之前打不过的怪，现在能打过

**案例：暗黑破坏神**
- 满屏的伤害数字提供即时的强度反馈
- 从几十到几万的数值膨胀让玩家明确感知成长

### 2.2 选择的意义

**问题**：如果成长路径唯一，玩家只是按按钮。

**设计方法**：
- **分支选择**：技能树的分支（无法点满所有）
- **资源取舍**：有限资源下选择强化哪个角色/装备
- **Build构建**：不同装备组合产生不同玩法

**案例：流放之路（POE）**
- 庞大的天赋树让玩家必须做出取舍
- 同一职业有数十种 viable Build

### 2.3 节奏控制

**问题**：成长太快会毕业弃坑，太慢会流失。

**设计公式**：

```
成长速度 = 内容消耗速度 × 系数(1.2-1.5)
```

即：玩家总是"差不多能跟上"，但永远"差一点毕业"。

**关键节点**：
- **新手期**（前7天）：快速升级，建立成就感
- **成长期**（第1个月）：稳定成长，建立习惯
- **成熟期**（后期）：边际收益递减，转向社交/收集

## 三、防毕业机制

### 3.1 为什么要防毕业？

- **内容生产速度 << 内容消耗速度**
- 玩家毕业后失去目标，容易流失
- 需要创造"永远差一点"的心理状态

### 3.2 常见防毕业设计

#### A. 随机性防线（最常用）

**原理**：通过随机性延长"完美"的获取时间。

| 设计 | 案例 |
|------|------|
| 装备词条随机 | 原神圣遗物 |
| 掉落随机 | 暗黑破坏神 |
| 强化成功率 | 传统MMO的+10装备 |

**风险**：挫败感强，可能导致玩家弃坑。

#### B. 软上限设计

**原理**：设置多个"毕业标准"，每个都有边际收益。

**案例：星穹铁道的行迹系统**
- 可以点到10级
- 但8级之后每级消耗巨大，收益递减
- 让玩家自己决定"够用"的标准

#### C. 横向扩展

**原理**：不是让一个角色更强，而是让更多角色可用。

**案例：原神的深渊系统**
- 需要2支队伍（8个角色）
- 即使主C毕业，还需要培养副C、辅助
- 横向收集延长游戏寿命

#### D. 动态难度

**原理**：成长后内容也变难，保持挑战。

**案例：暗黑破坏神的大秘境**
- 层数无限上升
- 装备越好，能打更高层
- 永远有下一个目标

### 3.3 防毕业的度

| 程度 | 结果 | 案例 |
|------|------|------|
| **太松** | 快速毕业→流失 | 某些单机DLC |
| **适中** | 持续追求→留存 | 星铁、原神 |
| **太紧** | 挫败感→流失 | 某些韩式MMO的强化失败 |

## 四、案例分析

### 4.1 圣遗物系统：随机的极致

👉 [详细分析见案例库](../99-案例库/原神圣遗物系统.md)

**核心启示**：通过分层随机（主词条保底+副词条彩票）和沉没成本陷阱，创造了几乎无限的长线目标。

### 4.2 暗黑破坏神系列：装备驱动鼻祖

**设计循环**：
```
刷怪 → 掉装备 → 变强 → 刷更强的怪 → 掉更好的装备
```

**关键设计**：
- **Build构建**：不同装备组合改变玩法
- **大秘境**：无限难度提供无限目标
- **赛季制**：定期重置，所有人重新开始

## 五、设计模板

### 模板1：成长曲线验证

```
1. 画出玩家强度随时间的变化曲线
2. 检查：
   □ 新手期是否有陡峭上升（建立成就感）
   □ 中期是否稳定成长（保持动力）
   □ 后期是否有明确目标（防毕业）
3. 测试：一个纯新玩家要多久能达到"当前版本平均水平"？
4. 确保：这个时间是可接受的（太久了会劝退新玩家）
```

### 模板2：防毕业机制检查

```
1. 列出所有毕业标准（什么算"毕业"）
2. 计算：纯肝需要多久能达到？
3. 计算：中氪玩家需要多久？
4. 计算：重氪玩家需要多久？
5. 检查：
   □ 是否有"永远无法毕业"的随机因素？
   □ 新玩家追赶机制是否合理？
   □ 是否有横向扩展空间（多角色/多Build）？
```

### 模板3：成长反馈密度检查

```
1. 列出玩家每次登录能获得的所有成长
2. 分类：
   □ 即时反馈（立刻感知）
   □ 短期反馈（本次游戏内）
   □ 长期反馈（跨多次游戏）
3. 确保：
   □ 每次登录都有至少一个即时反馈
   □ 每次游戏都有至少一个短期目标达成
   □ 长期目标分解为可感知的里程碑
```

## 六、常见误区

### ❌ 误区1：只有数值成长
只给攻击力+10，不给新技能/新机制。

**问题**：枯燥，玩家感觉自己在"按计算器"。

### ❌ 误区2：成长不可见
角色变强了，但玩家感知不到。

**问题**：失去成长的核心驱动力。

### ❌ 误区3：防毕业过度
随机性太强或要求太高，玩家感到绝望。

**问题**：挫败感导致流失。
